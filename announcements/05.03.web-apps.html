Web Apps and Discord

This week many of you are discovering the primary reason we do not use a printed textbook for navigating cloud service providers' platforms in this course.

On-line and on-demand services are constantly changing. New features are added, existing features are modified, and some features are removed entirely. And interfaces are updated frequently. This is the nature of technology in the 21st century.

The fundamentals apply.

The first thing many of you asked about is the development stack. As most of you probably already know, the available LTS (Long Term Support) version of .NET will change over time. This year, .NET 8 is what is supported on Azure App Services.

That's fine. I did build and run the provided code on the more recent .NET version, and there were no issues. No modifications to the code were necessary. Just like the options though, some of the deployment steps and details may change over time. Applying the concepts, the steps worked fine and the demonstation was successful.

Sometimes things like the deployment region are impacted, so by selecting a different, less busy region, you options and costs may be different. You may be able to host a web app or storage account in Canada for instance, but not in the US East region, for a particular period in time. These things do change, but the concepts remain the same.
